1 Resumen Ejecutivo

Se construyó una plataforma SaaS completa para automatización de WhatsApp llamada WhatsBot. El sistema permite a los usuarios:

  • Conectar su cuenta de WhatsApp mediante código QR
  • Enviar y recibir mensajes en tiempo real
  • Gestionar contactos con tags y búsqueda
  • Configurar respuestas automáticas basadas en palabras clave
  • Ver estadísticas de uso desde un dashboard profesional

La arquitectura es un monorepo con separación clara entre frontend, backend, motor de WhatsApp y base de datos.

5 Base de Datos — 5 Modelos

ModeloCampos claveRelaciones
User id, email, password, name, plan → Sessions, Contacts, Automations
WhatsAppSession id, userId, sessionName, status, qrCode, phone → User, Messages
Contact id, userId, phone, name, tags (JSON), notes → User, Messages
Message id, sessionId, contactId, direction, body, status → Session, Contact
Automation id, userId, name, trigger, response, matchType, enabled → User
Valores posibles:
• Plan: FREE, STARTER, PRO, ENTERPRISE
• Status sesión: CONNECTED, DISCONNECTED, QR_READY
• Direction: INBOUND, OUTBOUND
• Status msg: PENDING, SENT, DELIVERED, READ, FAILED
• MatchType: EXACT, CONTAINS, STARTS_WITH, REGEX

7 Eventos en Tiempo Real (Socket.IO)

  Usuario crea sesión
       │
       ▼
  Motor WA genera QR ─────► Socket.IO ─────► Dashboard muestra QR
       │
  Usuario escanea QR con teléfono
       │
       ▼
  Motor WA → "ready" ─────► Socket.IO ─────► Dashboard: "Conectado ✅"
       │
  Alguien envía un mensaje al WhatsApp del usuario
       │
       ▼
  Motor WA → filtra (no broadcast/grupos)
       │
       ▼
  API: Crea contacto si no existe
  API: Guarda mensaje en DB
  API: Verifica automatizaciones
       │
       ├── Si hay match → envía auto-respuesta
       │
       ▼
  Socket.IO ─────► Dashboard: mensaje aparece en chat

11 Flujo Principal del Usuario

1
El usuario abre http://localhost:3000 y ve la landing page
2
Se registra con email y contraseña
3
Entra al Dashboard → ve stats vacíos y checklist de inicio
4
Va a WhatsApp → crea una sesión → aparece el código QR
5
Escanea el QR con su teléfono → sesión se marca como "Conectado"
6
Cuando alguien le escribe, el mensaje aparece en tiempo real en Conversaciones
7
Si hay automatización configurada con match, se envía auto-respuesta
8
Los contactos se crean automáticamente al recibir mensajes

12 Cómo Iniciar el Proyecto

# Terminal 1 — Backend API
cd apps/api
npm run dev
# → http://localhost:3001

# Terminal 2 — Frontend
cd apps/web
npm run dev
# → http://localhost:3000

# (Opcional) Terminal 3 — Worker
cd apps/worker
npm run dev
⚠️ Importante: No se necesita Docker para desarrollo. La base de datos SQLite se crea automáticamente en apps/api/prisma/dev.db.

14 Decisiones Técnicas

DecisiónRazón
Express.js en vez de NestJSMás rápido para MVP, menor complejidad inicial
SQLite en vez de PostgreSQLSin dependencia de Docker; migración trivial
Zustand en vez de ReduxAPI simple, ligero, perfecto para esta escala
Socket.IO para real-timeMayor compatibilidad, maneja reconexiones auto
Monorepo simplePackages compartidos sin overhead de Turborepo
Strings en vez de EnumsSQLite no soporta enums; validación en app
Tags como JSONSQLite no tiene arrays nativos; JSON.parse()
🔧 Para agregar funcionalidades:
• Nuevas rutas API: Crear en apps/api/src/modules/[nombre]/ → registrar en index.js
• Nuevas páginas: Crear en apps/web/src/app/dashboard/[nombre]/page.tsx
• Nuevos modelos: Editar prisma/schema.prisma → ejecutar npx prisma db push
• Nuevos eventos WA: Editar packages/whatsapp/index.js + whatsapp.socket.js
